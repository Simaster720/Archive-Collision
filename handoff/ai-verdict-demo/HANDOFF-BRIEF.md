# 핸드오프 지시문 — AI 판별 박스 디자인 수정

> **사용법**: 디자인 수정 세션을 시작할 때, 이 문서 전체를 코딩 에이전트(또는 함께 작업하는
> 사람)에게 `index.html`과 함께 그대로 붙여넣어 주세요. CONFIG가 무엇이고, 무엇을 만지고
> 무엇을 만지지 말아야 하는지에 대한 맥락을 한 번에 전달합니다.

---

## 📋 붙여넣기용 지시문 (여기부터 복사)

당신은 `handoff/ai-verdict-demo/index.html` 한 파일의 **AI 판별 트래킹 박스 시각 디자인**을
다듬는 작업을 돕습니다. 아래 맥락과 규칙을 반드시 지켜 주세요.

### 1) 이 작업이 무엇인가
- "신수찬 컬렉션" 아카이브 웹사이트는 미술관 아카이브 UI를 모사하되, 각 자료 이미지 위에
  가짜 **"AI 판별"** 오버레이(색상 트래킹 박스 + 의심도 %)를 씌워 *AI 판별의 정치성*을
  드러내는 작업입니다.
- **개념의 핵심**: 데모의 5장은 전부 **실제로 찍은 진짜 사진**입니다. 그런데도 박스가
  빨갛게 뜨며 "AI로 의심됨"이라 말하는 것(= "오답")이 작업의 핵심입니다. 진실된 자료를
  관객이 의심하게 만드는 긴장이 디자인의 목표이지, 단순히 "예쁜 박스"가 목표가 아닙니다.
  (5장 중 4번 `S1_SS8_02`만 "정답"이라 초록으로 평온합니다 — 대비용입니다.)

### 2) 무엇을 만지나 (★ 가장 중요한 규칙)
- `index.html` 안에는 두 영역이 있습니다:
  - **`CONFIG = { … }` 블록** = 디자인 면. **여기 값만** 바꾸세요.
  - 그 아래 **`ENGINE`** = 렌더 로직. 작가는 손대지 않습니다.
- **작가 단독 작업이면**: 오직 CONFIG 값만 수정합니다.
- **코딩 에이전트가 함께라면**: 새 표현이 필요할 때 ENGINE을 확장해 **새 knob을 추가**해도
  됩니다 — 단, 아래 "이식성 계약"을 반드시 지켜야 합니다.

### 3) 이식성 계약 (이 파일의 존재 이유)
- 이 데모는 배포되지 않습니다(`app/` 밖). 여기서 확정한 **CONFIG 값만** 나중에 실제 웹
  (Next.js / React / Tailwind)으로 1:1 이식됩니다. 그래서 **CONFIG가 곧 결과물**입니다.
- 따라서:
  - 디자인 변경은 가능한 한 **CONFIG 값 조정**으로 표현하세요.
  - 구조를 직접 뜯어고치기보다 **새 knob 추가**를 우선하세요. knob을 추가할 땐:
    1. CONFIG에 항목 + 한 줄 주석 추가,
    2. ENGINE에서 그 knob을 읽어 동작하도록 배선,
    3. **기본값은 "지금 모습 그대로"**가 되도록(다른 5장 회귀 방지),
    4. React 속성/CSS 변수에 그대로 대응 가능한 단순 값(숫자·문자열·불리언·색 배열)으로.
  - `data.js`는 자동 생성물이니 **수정 금지**. 파일을 `app/`로 옮기지 마세요.
  - 색 의미 체계(빨강≥0.75 / 노랑≥0.50 / 초록<0.50)는 데이터 규약이므로, 바꾸려면
    `colorScale` knob을 통해서만(임계값·색 교체) 바꾸세요.

### 4) 데이터 현실 (알고 있어야 박스 큐레이션이 됨)
- 박스 1개 = 판별기의 "평가항목" 1개(한글 특징명, 예: "헤일로 아티팩트"). 각 박스는
  `번호 · 평가항목명 · 의심도%`와 색을 가집니다.
- 박스 색 = AI 의심 강도: 🔴 빨강(≥75%) / 🟡 노랑(≥50%) / 🟢 초록(<50%).
- 데이터엔 이미지당 **박스가 35개**나 있고 대부분이 전체화면 영역입니다. 그래서 기본은
  **신호 강한 소수만** 추려 보여줍니다(`boxCount` + `filter:'signal'`). 점수 없는 박스는
  기본 건너뜁니다(`skipNullScore`).
- 박스 좌표는 **원본 픽셀** 기준이며 표시 크기에 맞춰 %로 스케일됩니다(직접 px 계산 불필요).

### 5) 작업·검증 방법
- `index.html`을 브라우저로 열어 둔 채(인터넷 연결 필요 — 이미지는 Cloudinary) CONFIG를
  고치고 **새로고침**하며 반복합니다. 빌드 과정 없음.
- 변경이 **5장 전부에서** 무너지지 않는지 확인하세요. 일부러 다양하게 골랐습니다:
  - `S3_SS2_05` 초광각(3.05) · 전부 빨강 = 최대 경보
  - `S1_SS1_01` 정사각(2048²)
  - `S2_SS2_04` 세로(1080×1350)
  - `S1_SS8_02` 4K 가로 · 유일한 초록/평온
  - `S3_SS3_34` 소형(710×531) · 작은 원본에서의 가독성
- 좁은 화면(모바일) 반응형도 함께 보세요(사이드리스트가 아래로 내려갑니다).

### 6) 다 하면 — 반환
- 수정한 **`index.html`을 그대로 돌려주세요**(knob을 추가했다면 그 ENGINE 변경 포함).
- 개발 쪽에서 **CONFIG 값**(+ 추가된 knob의 의미)을 복사해 웹 컴포넌트로 이식합니다.
- 망가지면: `data.js`는 그대로 두고 CONFIG를 아래 기본값으로 되돌리면 복구됩니다.

## 🎛 CONFIG 손잡이(knob) 전체 — 기본값과 의미

**선별 (박스 몇 개·어떤 기준)**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `boxCount` | `6` | 노출 박스 수 (`showAll:true`면 무시) |
| `filter` | `'signal'` | `'signal'` 신호 강한 순 · `'partial'` 전체화면 박스 제외 · `'all'` 전부 |
| `showAll` | `false` | `true`면 35개 전부(데이터 전체 확인용) |
| `sortBy` | `'score'` | `'score'` 의심도 높은 순 · `'order'` 데이터 원래 순서 |
| `skipNullScore` | `true` | 점수 없는 박스 건너뛰기(`false`면 회색) |

**박스 모양**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `borderWidth` | `2` | 테두리 두께(px) |
| `borderStyle` | `'solid'` | `'solid'` · `'dashed'` · `'dotted'` |
| `borderRadius` | `0` | 모서리 둥글기(px) |
| `cornerStyle` | `'plain'` | `'plain'` 전체 테두리 · `'brackets'` 좌상·우하 ⌐ · `'corners'` 네 모서리 |
| `fillOpacity` | `0` | 내부 채움 0~1 (0.08쯤 = 은은한 하이라이트) |

**색**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `useDataColor` | `true` | `true` 데이터 원본 색 · `false` `colorScale`로 재계산 |
| `colorScale` | 빨강.75/노랑.50/초록0 | `useDataColor:false`일 때 임계값→색. 빨강 기준을 낮추면 더 많이 빨개짐 |

**라벨 (박스에 붙는 글자)**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `showNumber` `showTitle` `showPct` | 모두 `true` | `#1` · 평가항목명 · `%` 각각 토글 |
| `labelPosition` | `'inside-tl'` | `'inside-tl'` 박스 안 좌상단 · `'outside'` 박스 위 · `'none'` 숨김 |
| `labelSize` | `11` | 라벨 글자 크기(px) |

**근거 표시**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `evidenceMode` | `'sidelist'` | `'sidelist'` 옆 목록 · `'tooltip'` 박스 hover · `'caption'` 이미지 아래 한 줄 |
| `sidelistPosition` | `'right'` | `'right'` 오른쪽 · `'below'` 아래 2열 |
| `showDescription` | `true` | 옆 목록에 근거 설명 문장 표시 |

**종합 문구**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `showSummary` | `true` | 종합 문구 표시 |
| `summaryTemplate` | `'{pct}% 정도 생성된 것으로 의심됩니다'` | 치환자: `{pct}` `{id}` `{answer}` `{judgement}` `{algo}` `{gpt}` |
| `summaryPosition` | `'top'` | `'top'` 이미지 위 · `'bottom'` 아래 |
| `showCardHeader` | `true` | id·원본크기·판정 배지 줄(데모 맥락용) |

**연출 (확정 대상 아님 · 탐색용 · 기본 off)**
| knob | 기본 | 값 / 의미 |
|---|---|---|
| `enableAnimation` | `false` | 종합 문구 타이프라이터 + 박스 순차 등장 |
| `typeSpeed` `revealStagger` | `28` `90` | 타이핑 속도 · 박스 등장 간격(ms) |
| `enableJitter` | `false` | 박스 미세 떨림("탐지 중" 느낌) |
| `jitterAmount` | `3` | 떨림 폭(px) |

## 🍳 자주 쓰는 조정 레시피 (예시)

- **"경보를 더 세게"** → `borderWidth: 3`, `fillOpacity: 0.08`, `colorScale` 빨강 기준
  `0.75 → 0.6`(`useDataColor: false`와 함께).
- **"박스를 더 적게/깔끔하게"** → `boxCount: 3`, `filter: 'partial'`.
- **"근거를 이미지 안에서 바로"** → `evidenceMode: 'tooltip'` 또는 `'caption'`.
- **"라벨이 복잡하다"** → `showTitle: false`(번호+%만) 또는 `labelPosition: 'outside'`.
- **"실시간 분석되는 느낌"** → `enableAnimation: true`(필요시 `enableJitter: true`).
- **"전체 데이터가 궁금"** → `showAll: true`(35개 전부, 임시 확인용).

> 새 아이디어가 위 knob으로 표현되지 않으면, 구조를 바꾸기 전에 **"이런 knob을 추가해
> 달라"**고 요청하세요(예: `labelPosition`에 `'below'` 추가, `boxShadow` knob 추가). 그래야
> 웹 이식이 깨지지 않습니다.
