# AI 판별 트래킹 박스 — 작가 핸드오프 데모

> AI 판별 오버레이(이미지 위 색상 **트래킹 박스**)의 **시각 디자인을 확정**하기 위한 단독 데모입니다.
> 실제 판별 데이터 5장으로 박스를 그려 놓았으니, 박스의 모양·색·라벨·근거 표시를 직접 만져보고
> 마음에 드는 값으로 맞춰 주세요. 여기서 확정한 값만 나중에 실제 웹사이트로 옮깁니다.

---

## 0. 여는 법

이 폴더의 **`index.html` 을 브라우저로 그냥 더블클릭**해서 열면 됩니다.
(인터넷 연결이 필요합니다 — 이미지는 Cloudinary 서버에서 불러옵니다.)

```
handoff/ai-verdict-demo/
├─ index.html   ← 이 파일을 브라우저로 연다 + 여기서 디자인을 수정한다
├─ data.js      ← 5장의 판별 데이터 (자동 생성물, 손대지 않음)
└─ README.md    ← 지금 읽는 안내
```

---

## 1. 이게 무엇이고, 무엇이 아닌가

- **이다**: 박스 디자인을 눈으로 보며 확정하는 작업판. 5장을 한 화면에 세로로 나열했고,
  디자인을 한 번 고치면 5장에 동시에 반영됩니다.
- **아니다**: 실제 배포되는 웹사이트가 아닙니다. 이 파일은 `app/` 밖(`handoff/`)에 있어
  배포에 **포함되지 않습니다.** 그래서 여기서 중요한 건 "최종 결과물"이 아니라 **확정된 값**입니다.

### 왜 이 5장인가
방향·해상도·의심도·색 분포를 최대한 다양하게 골랐습니다. 디자인이 어떤 이미지에서도
무너지지 않는지 한 번에 확인하려는 의도입니다.

| # | 이미지 | 크기 | 특징 |
|---|--------|------|------|
| 1 | `S3_SS2_05` | 1006×330 (초광각) | 최고 의심도(80%)·전부 빨강 = **최대 경보** |
| 2 | `S1_SS1_01` | 2048×2048 (정사각) | 균형 잡힌 색 |
| 3 | `S2_SS2_04` | 1080×1350 (세로) | 노랑 우세 |
| 4 | `S1_SS8_02` | 4032×3024 (4K 가로) | **유일한 "평온" 상태** — 초록·낮은 의심도(8%) |
| 5 | `S3_SS3_34` | 710×531 (소형) | 작은 원본에서의 박스 가독성 |

---

## 2. 데이터 한눈에

- 박스 하나 = 판별기가 본 "평가항목" 하나. 각 박스는 **번호 · 평가항목 이름 · 의심도(%)**를 가집니다.
- **박스 색 = AI 의심 강도**:
  - 🔴 **빨강** (≥75%) — AI 신호 강함
  - 🟡 **노랑** (≥50%) — 중간 신호
  - 🟢 **초록** (<50%) — 실제 사진 특성
- 데이터엔 이미지당 **35개 박스**가 있는데 대부분이 전체화면 영역이라, 기본값은
  **신호가 강한 소수만** 추려서 보여줍니다(`boxCount`·`filter`로 조절).
- 점수가 없는(검출 불가) 박스가 있을 수 있어 기본은 **건너뜁니다**(`skipNullScore`).

> 💡 **개념의 핵심**: 이 5장은 전부 **진짜로 찍은 사진**입니다. "오답" 배지가 붙은 4장은
> 진짜 사진을 AI 생성물로 **오인**한 것 — 관객이 진실된 자료를 의심하게 만드는 게 작업의 목적입니다.
> (4번만 "정답"이라 초록으로 평온합니다.)

## 3. 어떻게 수정하나 (★ 가장 중요)

`index.html` 을 텍스트 에디터(또는 코딩 에이전트)로 열면 아래쪽에 이런 블록이 있습니다:

```js
const CONFIG = {
  boxCount: 6,
  filter: 'signal',
  ...
};
```

**이 `CONFIG = { … }` 블록 안에서만** 숫자나 단어를 바꾸세요.
그 아래 "ENGINE"이라고 표시된 코드는 **건드리지 않아도 됩니다.**
저장한 뒤 브라우저를 새로고침하면 5장에 바로 반영됩니다.

### CONFIG 손잡이(knob) 설명

**선별 — 박스 몇 개를 어떤 기준으로 보여줄지**
| knob | 값 | 뜻 |
|------|----|----|
| `boxCount` | 숫자 | 노출 박스 수 (기본 6) |
| `filter` | `'signal'` / `'partial'` / `'all'` | 신호 강한 순 / 전체화면 박스 제외 / 전부 |
| `showAll` | `true` / `false` | true면 35개 전부 표시(데이터 전체 확인용) |
| `sortBy` | `'score'` / `'order'` | 의심도 높은 순 / 데이터 원래 순서 |
| `skipNullScore` | `true` / `false` | 점수 없는 박스 건너뛰기 / 회색 표시 |

**박스 모양**
| knob | 값 | 뜻 |
|------|----|----|
| `borderWidth` | 숫자(px) | 테두리 두께 |
| `borderStyle` | `'solid'` / `'dashed'` / `'dotted'` | 테두리 스타일 |
| `borderRadius` | 숫자(px) | 모서리 둥글기 |
| `cornerStyle` | `'plain'` / `'brackets'` / `'corners'` | 전체 테두리 / 좌상·우하 ⌐ / 네 모서리 |
| `fillOpacity` | 0 ~ 1 | 박스 내부 채움 (0=투명, 0.08=은은한 하이라이트) |

**색**
| knob | 값 | 뜻 |
|------|----|----|
| `useDataColor` | `true` / `false` | 데이터 원본 색 사용 / `colorScale`로 재계산 |
| `colorScale` | 배열 | `useDataColor:false`일 때 임계값→색 (예: 빨강 기준을 0.6으로 낮추기) |

**라벨 (박스에 붙는 글자)**
| knob | 값 | 뜻 |
|------|----|----|
| `showNumber` `showTitle` `showPct` | `true`/`false` | #번호 / 평가항목 이름 / % 각각 켜고 끄기 |
| `labelPosition` | `'inside-tl'` / `'outside'` / `'none'` | 박스 안 좌상단 / 박스 위 / 숨김 |
| `labelSize` | 숫자(px) | 라벨 글자 크기 |

**근거 표시 방식**
| knob | 값 | 뜻 |
|------|----|----|
| `evidenceMode` | `'sidelist'` / `'tooltip'` / `'caption'` | 옆 목록 / 박스에 마우스 올리면 / 이미지 아래 한 줄 |
| `sidelistPosition` | `'right'` / `'below'` | 이미지 오른쪽 / 이미지 아래(2열) |
| `showDescription` | `true` / `false` | 옆 목록에 근거 설명 문장 표시 |

**종합 문구**
| knob | 값 | 뜻 |
|------|----|----|
| `showSummary` | `true` / `false` | 종합 문구 표시 |
| `summaryTemplate` | 문자열 | 문구 틀. 치환자: `{pct}` `{id}` `{answer}` `{judgement}` `{algo}` `{gpt}` |
| `summaryPosition` | `'top'` / `'bottom'` | 이미지 위 / 아래 |

**연출 (확정 대상 아님 · 탐색용 · 기본 꺼짐)**
| knob | 값 | 뜻 |
|------|----|----|
| `enableAnimation` | `true` / `false` | 종합 문구 타이프라이터 + 박스 순차 등장 |
| `typeSpeed` `revealStagger` | 숫자(ms) | 타이핑 속도 / 박스 등장 간격 |
| `enableJitter` | `true` / `false` | 박스 미세 떨림("탐지 중" 느낌) |
| `jitterAmount` | 숫자(px) | 떨림 폭 |

> 연출 두 가지는 "탐색용"입니다. 정밀 타이밍·안무는 별도 작업이므로 여기서 확정하지 않아도 됩니다.
> 켜서 느낌만 봐 주세요. (시스템 "동작 줄이기" 설정이 켜져 있으면 자동으로 꺼집니다.)

## 4. 다 하면 — 돌려주는 법

수정한 **`index.html` 파일을 그대로 돌려주세요.** 개발 쪽에서 CONFIG 값만 복사해
실제 웹(React/Tailwind)으로 1:1 옮깁니다. CONFIG의 모든 손잡이는 웹 컴포넌트의
속성/CSS 변수에 그대로 대응되도록 설계돼 있습니다.

- 값을 바꾸다 망가지면: `data.js`는 그대로 두고, CONFIG를 위 표의 기본값으로 되돌리면 됩니다.
- **새 아이디어**(예: "박스에 그림자를 넣고 싶다", "라벨을 박스 아래에 두고 싶다")가 있으면,
  구조를 직접 뜯어고치기보다 **"이런 knob을 추가해 달라"고 요청**해 주세요. 그래야 웹 이식이
  깨지지 않습니다. (예: `labelPosition`에 `'below'` 추가, `boxShadow` knob 추가 등)

## 5. (개발용) 데이터 재생성

`data.js`는 아래 스크립트로 `eval_results.xlsx`에서 1회 생성된 자동 산출물입니다.
**작가 작업 중에는 다시 돌릴 필요가 없습니다**(돌리면 `data.js`만 새로 쓰고, `index.html`은
건드리지 않습니다 — 디자인 작업은 안전합니다).

```bash
node scripts/extract-ai-demo-data.mjs
```

- 입력: `data/source/eval_results.xlsx` (시트 `평가결과`)
- 출력: `handoff/ai-verdict-demo/data.js` (`window.AI_DEMO_DATA`)
- 대상 5장은 스크립트 상단 `TARGETS` 배열에 등록번호로 지정돼 있습니다.
- 이미지는 Cloudinary 딜리버리 URL(`res.cloudinary.com/<cloud>/image/upload/...`)로 불러오며,
  박스 좌표는 데이터의 **원본 픽셀**(`image.width/height`) 기준이라 표시 크기와 무관하게
  정확히 따라옵니다.

## 6. 용어 (프로젝트 공통)

- **트래킹 박스** = 바운딩 박스. AI 판별 시 이미지 위에 그려지는 색상 사각형.
- **평가항목** = 박스가 가리키는 한글 특징명(예: "FFT 주파수 분포", "헤일로 아티팩트").
- **AI 판별** = 이미지가 몇 % 생성됐는지 판정하는 분석. 최종점수 ≥ 50이면 "AI 판정".
- **오답** = 진짜 사진을 AI로 오인한 사건(이 작업의 핵심).

자세한 용어는 레포 루트의 [`CONTEXT.md`](../../CONTEXT.md), 데이터 형식은
[`data/source/출력형식_설명.txt`](../../data/source/출력형식_설명.txt) 참고.
