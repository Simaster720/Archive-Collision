1. 전체 구조 (eval_results.json)
================================================================

{
  "total": 200,          // 총 이미지 수
  "correct": 150,        // 정답 수
  "accuracy": 75.0,      // 정확도 (%)
  "results": [ ... ]     // 이미지별 결과 배열
}


================================================================
2. results 배열 — 이미지 1개당 구조
================================================================

{
  "파일명": "image01.png",

  "정답": "AI" | "실제",        // 실제 라벨 (파일명 규칙으로 결정)

  "알고리즘점수": 91,            // 알고리즘 단독 점수 (0~100)
  "GPT앙상블점수": 21,           // GPT/Claude 리전 분석 점수 (0~100)
  "최종점수": 56,                // 최종 점수 = (알고리즘 + GPT) / 2  (0~100)
                                 //   50 이상이면 AI 판정
                                 //   50 미만이면 실제 사진 판정

  "판정": "정답" | "오답",       // 모델 판정이 정답 라벨과 일치하는지

  "GPT판정": "AI 생성 가능성 높음",   // GPT/Claude가 내린 한 줄 결론
  "GPT상세설명": "...",               // GPT/Claude 상세 분석 텍스트

  "특징JSON": { ... }            // ← HTML 렌더링에 핵심! 아래 3번 참고
}


================================================================
3. 특징JSON — 박스 렌더링용 핵심 데이터
================================================================

{
  "image": {
    "width": 507,          // 원본 이미지 가로 픽셀
    "height": 701          // 원본 이미지 세로 픽셀
  },

  "algo_score": 91,        // 이미지 전체 알고리즘 점수 (0~100)
  "gpt_score":  21,        // 이미지 전체 GPT 점수 (0~100)
  "final_score": 56,       // 최종 점수 (0~100)

  "features": [            // 35개 특징 배열 (각각 박스 하나)
    {
      "title": "FFT 주파수 분포",       // 한글 특징명 (박스 라벨용)
      "key":   "fft_score",             // 영문 키

      "x1": 0,    "y1": 0,             // 박스 좌상단 좌표 (픽셀)
      "x2": 507,  "y2": 701,           // 박스 우하단 좌표 (픽셀)
                                        // → 이 좌표로 이미지 위에 박스 그리기

      "algo_score":     0.84,           // 알고리즘 점수 (0.0~1.0)
      "gpt_score":      0.20,           // GPT 점수     (0.0~1.0)
      "ensemble_score": 0.52,           // 앙상블 점수  (0.0~1.0)  ← 박스 색 결정에 사용
      "score_pct":      52,             // ensemble_score × 100 (0~100 정수)

      "color": "#F9E2AF",               // 박스 색상 (아래 색상표 참고)

      "description": "주파수 분포가 AI 생성 패턴과 유사 ..."
                                        // 박스 툴팁/설명용 텍스트
    },
    ...
  ]
}


================================================================
4. 박스 색상 기준 (color 필드)
================================================================

  "#F38BA8"  (빨강)  → ensemble_score >= 0.75  : AI 신호 강함
  "#F9E2AF"  (노랑)  → ensemble_score >= 0.50  : 중간 신호
  "#A6E3A1"  (초록)  → ensemble_score <  0.50  : 실제 사진 특성

  ※ color 필드를 그대로 CSS border-color / background-color 에 사용 가능


================================================================
5. HTML 렌더링 예시 (박스 그리기 로직)
================================================================

// 이미지 위에 position: relative 컨테이너를 만들고
// features 배열을 순회하며 position: absolute 박스 생성

const scaleX = 표시할_이미지_가로 / image.width;
const scaleY = 표시할_이미지_세로 / image.height;

features.forEach(f => {
  const box = document.createElement('div');
  box.style.position   = 'absolute';
  box.style.left       = (f.x1 * scaleX) + 'px';
  box.style.top        = (f.y1 * scaleY) + 'px';
  box.style.width      = ((f.x2 - f.x1) * scaleX) + 'px';
  box.style.height     = ((f.y2 - f.y1) * scaleY) + 'px';
  box.style.border     = '2px solid ' + f.color;
  box.title            = f.title + ' (' + f.score_pct + '%) — ' + f.description;
});


================================================================
6. 주의사항
================================================================

- algo_score / gpt_score / ensemble_score 가 null 인 경우 있음
  → 해당 특징이 이미지에서 검출 불가한 경우 (예: 얼굴 없는 이미지의 face_score)
  → null 인 박스는 렌더링 스킵 또는 회색 처리 권장

- x1=0, y1=0, x2=이미지가로, y2=이미지세로 인 경우 = 전체 이미지 영역
  → 특정 위치를 못 잡은 특징 (전체를 스캔한 결과)

- GPT 크레딧 소진 시 gpt_score 는 algo_score 와 동일하게 들어감
  (LLM 없이 알고리즘만으로 계산된 결과)
