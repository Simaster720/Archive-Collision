# Archive Collision (아카이브 충돌)

"신수찬 컬렉션" 아카이브 웹사이트. 미술관 아카이브 UI를 모사하되, 각 자료 이미지 위에 가짜 "AI 판별" 오버레이를 씌워 'AI 판별의 정치성'을 드러내는 것이 작업의 개념적 핵심이다. (신승백김용훈의 작업 참조)

## Language

### 자료 계층 (Archive hierarchy)

**컬렉션 (Collection, [C])**:
아카이브 최상위 단위. 이 프로젝트엔 단 하나 — "신수찬 컬렉션". 사이드바 상단에 상시 고정되며 홈 버튼 역할을 한다.
_Avoid_: 아카이브(전체 시스템을 가리킬 때만), 갤러리

**시리즈 (Series, [S])**:
컬렉션 아래 첫 분류. 3개 존재 — 수업, 원우회, 교내활동.
_Avoid_: 카테고리, 폴더

**서브시리즈 (Sub-series, [SS])**:
시리즈 아래 분류. 파일들의 묶음. (예: "동기화된 감각과 물질", "디자인 워크샵")
_Avoid_: 하위폴더, 앨범

**파일 (File, [F])**:
계층의 말단. 실제 이미지 한 장 + 메타데이터 + AI 판별 결과. 목록에서는 이름이 아니라 정사각형 썸네일(원본 이미지)로 표시된다.
_Avoid_: 자료(맥락상 파일을 뜻할 때만), 항목, 이미지(이미지는 파일의 시각 자산만 지칭)

### 자료 메타데이터 (File metadata)

**등록번호 (Registration number)**:
파일의 고유 식별자이자 파일명. 계층을 인코딩한다 — `S{시리즈}_SS{서브시리즈}_{순번}.{확장자}` (예: `S1_SS1_01.png`). 실제 미술관 등록번호(예: MA-03-00005787)와는 별개 표기일 수 있음(미해결).

**자료내용 (Content description)**:
파일에 대한 서술. 길 수 있다. 메타데이터 5개 필드와 구분되는 본문 텍스트.

**AI 판별 (AI verdict)**:
파일 이미지가 "몇 % 생성(AI-generated)되었는지" 판정하는 가짜 분석 결과. 실시간 생성되는 듯 연출된다. 진실된 자료를 관객이 의심하게 만드는 것이 목적.

**트래킹 박스 (Tracking box)**:
AI 판별 시 이미지 위에 그려지는 색상 사각형(빨강/주황/초록 또는 파랑). 각 박스는 번호와 평가항목(예: 동작감지, 화질, 반복성)을 가지며, 같은 색의 판별 근거 문장과 짝지어진다. 매번 위치·크기에 약간의 랜덤성을 준다.

## Relationships

- 하나의 **컬렉션**은 여러 **시리즈**를 가진다 (현재 3개)
- 하나의 **시리즈**는 여러 **서브시리즈**를 가진다
- 하나의 **서브시리즈**는 여러 **파일**을 가진다
- 하나의 **파일**은 하나의 **이미지**, 메타데이터 5개 필드, 하나의 **AI 판별**을 가진다
- 하나의 **AI 판별**은 3~5개의 **트래킹 박스**와 종합 수치(%)로 구성된다

### UI 구성요소 (UI surfaces)

**사이드바 트리 (Sidebar tree)**:
좌측 고정 내비게이터. 컬렉션 헤더(상시 고정, 홈 버튼) → 시리즈 → 서브시리즈(펼치기/접기). SeMA 아카이브 분류 트리를 모사.

**메인 영역 (Main area)**:
우측 본문. 서브시리즈 선택 시 3열 정사각 썸네일 그리드, 파일 선택 시 상세(정보창).

**정보창 (Detail view)**:
선택된 파일의 상세. 서브시리즈명 → [F] 제목 → 메타데이터 2열 블록 → 자료내용 → (예약) AI 판별 영역.

**컬렉션 표지 (Collection cover)**:
첫 진입·홈 클릭 시 메인 영역에 뜨는 랜딩. 컬렉션 제목 중심의 미니멀 표지(기획서에 소개 문구가 없어 별도 카피는 넣지 않음).

## Flagged ambiguities

- **"자료"** — 파일을 뜻할 때도, 컬렉션 전체를 뜻할 때도 쓰임. 파일 단위를 가리킬 땐 **파일**로 통일.
- **등록번호 vs 파일명** — 엑셀에서 동일 열(`등록번호(파일명)`). 미술관 실제 등록번호(MA-…)와 코드형 파일명(S1_SS1_01)이 다를 수 있는지 미해결.
- **[C] 헤더의 이중 역할** — 사이드바 최상단 컬렉션 표기이자, (전역 헤더를 제외하므로) 홈 버튼. SeMA 트리에는 없는 동작.
